HTML Assets Localizer

一个强大的工具，用于将 HTML 文件中的外部资源（JavaScript、CSS 等）本地化， 同时支持 Node.js 命令行与浏览器界面，帮助你快速构建可靠的离线部署方案。

开始使用 查看源码

在线体验 - 本地化您的 HTML

上传包含外部资源的 HTML 文件，工具会自动下载远程 CSS/JS 资源、更新引用路径，并生成压缩包供下载，实现一键离线化。

支持自动解析 HTML 中的 script、 link 和 img 标签，并处理绝大多数常见的 CDN 资源。

目前支持引用外部 http(s) 资源的 HTML 文件。

满足离线部署、合规审查与本地调试等多种场景需求，让前端资源管理更安全、更可控。

智能识别 HTML 中的外部 JavaScript 和 CSS 资源，包含动态插入的常见 CDN 资源。

批量下载

安装与使用

使用 Node.js 版本可完成大型项目批量迁移，浏览器版本则适合快速预览或小型页面处理。

Node.js 版本

安装要求

  • Node.js 12+
  • 无需额外依赖，使用内置模块即可运行

使用方法

# 基本使用
node assets_localizer.js input.html target_directory

# 示例
node assets_localizer.js index.html myProject

浏览器界面

直接在本页面上传 HTML 文件，无需安装额外依赖，适合快速验证或临时处理资源。

处理完成后会生成压缩包，包含更新后的 HTML 与下载到本地的静态资源。

最佳实践

  • 在提交前使用 pnpm test 验证完整流程。
  • 本地化产物建议放入版本控制，确保环境一致。
  • 对大型资源库可结合缓存策略与断点续传进一步提速。

使用示例

输入 HTML 文件

假设页面引用了来自 CDN 的 Bootstrap 与 jQuery，我们希望将其拉取到本地。 

<!DOCTYPE html>
<html>
<head>
    <title>示例页面</title>
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/css/bootstrap.min.css">
    <script src="https://cdn.jsdelivr.net/npm/jquery@3.6.0/dist/jquery.min.js"></script>
</head>
<body>
    <h1>Hello World</h1>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

运行工具

使用 CLI 版本能够快速完成批量本地化： 

npx html-assets-localizer example.html ./output

输出结构

工具会生成包含 HTML 与所有静态资源的目录结构： 

目标目录/
├── index.html                # 处理后的HTML文件
├── js/                       # JavaScript文件目录
│   ├── jquery.min.js
│   ├── bootstrap.min.js
│   └── script_abc123.js
└── css/                      # CSS文件目录
    ├── bootstrap.min.css
    ├── style_def456.css
    └── main.css

作为模块使用

在 Node.js 项目中直接引入模块，可与现有业务流程无缝集成，结合自动化流程实现持续本地化。 

const HTMLScriptLocalizer = require('./assets_localizer');

// 创建实例
const localizer = new HTMLScriptLocalizer('input.html', 'target_directory');

// 处理文件
localizer.process();
