HTML Assets Localizer

自动解析并下载 HTML 引用的远程 JS/CSS，重写路径让页面脱离网络即可运行。 支持 CLI、可视化 UI 与浏览器端 SDK，多场景快速集成。

安装与开发

两步即可开始处理页面

根据团队使用场景选择命令行安装或本地开发流程，所有指令与 README 保持同步。

CLI 全局安装

全局安装后可在任何目录直接使用 `hal` 命令，也支持完整的 `html-assets-localizer`。 

pnpm add -g html-assets-localizer
npm install -g html-assets-localizer
yarn global add html-assets-localizer
# 快速处理
hal example.html ./offline-bundle

# 或使用全称命令
html-assets-localizer example.html ./offline-bundle
本地开发

克隆仓库后即可使用 `pnpm run dev` 进行命令行调试，提交前建议执行构建与测试。 

git clone https://github.com/YangsonHung/html-assets-localizer.git
cd html-assets-localizer
pnpm install
pnpm run dev

调试完成后运行 pnpm run build 输出到 dist/，并用 pnpm test 校验 Vitest 用例。

深入集成

在代码中使用 Localizer

同时支持 TypeScript / ESM 与 CommonJS，结合 README 中的架构说明可快速拓展工作流。

TypeScript / ESM 
import { HtmlAssetsLocalizer } from 'html-assets-localizer';

const localizer = new HtmlAssetsLocalizer({
  htmlFilePath: './example.html',
  targetDir: './offline',
});

const summary = await localizer.process();

console.log('输出路径:', summary.outputPath);
console.log('已下载资源数:', summary.assets.length);
CommonJS 
const { HtmlAssetsLocalizer } = require('html-assets-localizer');

async function run() {
  const localizer = new HtmlAssetsLocalizer({
    htmlFilePath: './example.html',
    targetDir: './offline',
  });

  const summary = await localizer.process();

  console.log('离线资源数量:', summary.assets.length);
  console.log('输出路径:', summary.outputPath);
}

run().catch(console.error);
